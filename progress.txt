## Running the Application

```bash
# Install dependencies
pnpm install

# Build shared package first
pnpm --filter @nonotion/shared build

# Start development servers
pnpm dev

# Or start individually
pnpm --filter @nonotion/api dev  # http://localhost:3001
pnpm --filter @nonotion/web dev  # http://localhost:5173
```

## Database Commands

```bash
# Generate new migration after schema changes
pnpm --filter @nonotion/api db:generate

# Apply migrations (also runs automatically on API startup)
pnpm --filter @nonotion/api db:migrate

# Open Drizzle Studio to inspect database
pnpm --filter @nonotion/api db:studio

# Migrate existing pages to multi-user (assigns to first admin)
pnpm --filter @nonotion/api migrate:multiuser
```

## Environment Variables

Create `apps/api/.env` based on `.env.example`:

```
JWT_SECRET=your-secure-secret-here
ADMIN_EMAIL=admin@example.com
```

## Running Tests

```bash
# E2E tests (requires dev servers)
pnpm --filter @nonotion/e2e test:e2e

# E2E with UI
pnpm --filter @nonotion/e2e test:e2e:ui
```

## Architecture Notes

### Phase 2 Changes

- Users and permissions stored in SQLite (data/nonotion.db)
- Pages and blocks remain in JSON files (data/pages/, data/blocks/)
- JWT tokens for authentication (stored in localStorage)
- First registered user (or ADMIN_EMAIL) becomes admin
- Pages have ownerId field for ownership tracking
- Three permission levels: owner (full control), editor (read/write), viewer (read only)

### Key New Files

| File | Purpose |
|------|---------|
| `apps/api/src/db/schema.ts` | Drizzle schema for users/permissions |
| `apps/api/src/storage/sqlite-storage.ts` | SQLite storage implementation |
| `apps/api/src/middleware/auth.ts` | JWT auth middleware |
| `apps/api/src/services/auth-service.ts` | Authentication logic |
