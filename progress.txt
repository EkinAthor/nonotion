## Running the Application

```bash
# Install dependencies
pnpm install

# Build shared package first
pnpm --filter @nonotion/shared build

# Start development servers
pnpm dev

# Or start individually
pnpm --filter @nonotion/api dev  # http://localhost:3001
pnpm --filter @nonotion/web dev  # http://localhost:5173
```

## Database Commands

```bash
# SQLite commands (default mode)
pnpm --filter @nonotion/api db:generate      # Generate migration
pnpm --filter @nonotion/api db:migrate       # Apply migrations
pnpm --filter @nonotion/api db:studio        # Open Drizzle Studio

# PostgreSQL commands
pnpm --filter @nonotion/api db:generate:pg   # Generate migration
pnpm --filter @nonotion/api db:migrate:pg    # Apply migrations
pnpm --filter @nonotion/api db:studio:pg     # Open Drizzle Studio

# Data migration
pnpm --filter @nonotion/api migrate:multiuser    # Migrate to multi-user
pnpm --filter @nonotion/api migrate:to-postgres  # Migrate to PostgreSQL
```

## Environment Variables

Create `.env` based on `.env.example`:

```bash
# Authentication
JWT_SECRET=your-secure-secret-here
ADMIN_EMAIL=admin@example.com

# Storage backend: 'json-sqlite' (default) or 'postgres'
STORAGE_TYPE=json-sqlite

# PostgreSQL URL (required when STORAGE_TYPE=postgres)
DATABASE_URL=postgresql://nonotion:nonotion@localhost:5432/nonotion
```

## Running Tests

```bash
# E2E tests (requires dev servers)
pnpm --filter @nonotion/e2e test:e2e

# E2E with UI
pnpm --filter @nonotion/e2e test:e2e:ui
```

## Architecture Notes

### Storage Backends

Nonotion supports two storage backends:

1. **json-sqlite** (default):
   - Pages and blocks stored in JSON files (data/pages/, data/blocks/)
   - Users and permissions stored in SQLite (data/nonotion.db)

2. **postgres**:
   - All data stored in PostgreSQL
   - Compatible with standard PostgreSQL and Supabase
   - Set `STORAGE_TYPE=postgres` and `DATABASE_URL` to enable

### Phase 2 Changes

- JWT tokens for authentication (stored in localStorage)
- First registered user (or ADMIN_EMAIL) becomes admin
- Pages have ownerId field for ownership tracking
- Three permission levels: owner (full control), editor (read/write), viewer (read only)
